# ForestRx 개선 방향 정리

## 1. 피벗 결론

ForestRx는 기존의 "개인 맞춤 등산 코스 추천 앱"에서 "안전 등급 기반 코스 추천 + 실시간 세이프 링크"를 제공하는 산행 안전 서비스로 피벗한다.

## 2. 기존 기획에서 유지할 것

- GPS 기반 주변 산/코스 추천
- 사용자 취향 입력: 이동 가능 시간, 등산 소요 시간
- 기상청 API 기반 강수량, 풍속, 기온 위험 분석
- 일몰 시간 기반 하산 가능성 판단
- 지도 기반 코스 위치/경로 시각화
- LLM 스타일의 자연어 추천 이유

## 3. 추천 서비스

### 보행 안전 산행 가이드

- 사용자는 선택하고 싶은 산까지 가능한 이동 시간과 등산 소요 시간을 간단히 입력한다.(1시간 단위로 설정)
- 시스템은 산코스별 추천점수와 안전점수를 계산한다. 단, 사용자가 선택한 시간은 절대값으로 자르지 않고 전후 버퍼를 적용한다.
- 결과 화면은 Top3 추천보다 "추천(그린)", "주의(노랑)", "비추천(빨강)"을 우선 노출한다.
- AI 에이전트는 추천 근거를 짧고 구체적으로 설명한다.
- 산행 중에는 실시간 세이프 링크로 보호자와 위치를 공유하고 보호자에게 알림을 보낸다. 다만 카카오맵 연동 방식에 따라 기능 범위가 달라지므로, 개발 착수 전 내부 지도 SDK 방식과 외부 링크 방식 중 하나를 결정한다.


## 4. 실시간 세이프 링크

Safe Link는 추천 이후 실제 산행 중 보호자와 보행 취약자를 연결하는 안전망 기능이다. 대회 관점에서는 AI 위험 예측이 화면 안내에 그치지 않고 실제 사고 예방 행동으로 이어진다는 점을 보여준다.

### B2C: 실시간 위치 공유 및 보호자 모니터링

- 동의 기반 위치 동기화: 아이, 노년층, 보행 약자 등 위치 공유 대상자가 앱에서 명시적으로 동의하면 보호자 단말기와 GPS 좌표를 동기화한다.
- 이동 경로 실시간 확인: 보호자는 지도 화면에서 대상자의 현재 위치와 이동 궤적을 확인한다.
- 예상 경로 이탈 감지: 추천된 안전 경로와 현재 이동 궤적이 일정 거리 이상 벗어나면 보호자에게 주의 알림을 보낸다.
- 위험 구역 진입 알림: 대상자가 AI가 분석한 보행 주의 구간, 사고 위험 구역, 급경사/암반/계곡 인접 구간에 진입하면 보호자 화면 또는 앱 알림으로 즉시 전달한다.
- 공유 종료/일시정지: 위치 공유자는 언제든 공유를 중단할 수 있어야 하며, 보호자 화면에도 공유 상태가 명확히 표시되어야 한다.

### 카카오맵 연동 기술 한계선

Safe Link는 카카오맵을 어떻게 연동하느냐에 따라 구현 가능한 기능이 달라진다. MVP 범위를 명확히 하기 위해 아래 두 가지 안 중 하나를 개발팀이 결정해야 한다.

#### A안: ForestRx 앱 내부에 Kakao Maps SDK 내장

- ForestRx 화면 안에서 지도, 현재 위치, 이동 궤적, 위험 구간을 직접 표시한다.
- 브라우저 Geolocation API 또는 앱 WebView 위치 권한으로 좌표를 주기적으로 수집한다.
- 백엔드가 좌표를 받아 경로 이탈, 위험 구역 진입, 마지막 위치 갱신 시각을 판단한다.
- Safe Link의 핵심인 실시간 보호자 모니터링과 위험 알림을 구현할 수 있다.
- 구현 난이도는 높지만 대회 출품 메시지에는 가장 적합하다.

#### B안: Kakao Map 외부 링크 연동

- 사용자를 카카오맵 앱 또는 외부 지도 링크로 이동시킨다.
- 경로 안내 경험은 빠르게 제공할 수 있지만, 사용자가 ForestRx 앱을 떠나므로 ForestRx 백엔드가 실시간 좌표를 안정적으로 받기 어렵다.
- 경로 이탈 감지, 위험 구역 진입 알림, 보호자 실시간 모니터링은 MVP에서 제외하거나 Mock 데모로만 표현해야 한다.
- 구현 난이도는 낮지만 Safe Link의 차별성이 약해진다.

#### 권장 결정

대회용 MVP에서는 A안을 권장한다. 단, 실제 푸시 알림까지 무리하게 구현하지 않고 앱 내부 알림 또는 보호자 화면 Mock 이벤트로 시연한다. B안을 선택할 경우 문서와 발표에서는 Safe Link를 "실시간 관제"가 아니라 "보호자 공유 링크"로 축소해 표현한다.

### UX 방향

- 산행 시작 버튼 이후 "세이프 링크 생성" CTA를 제공한다.
- 보호자는 링크 또는 초대 코드로 산행 모니터링 화면에 접근한다.
- 지도에는 현재 위치, 이동 궤적, 추천 경로, 위험 구간, 이탈 여부를 함께 표시한다.
- 알림 문구는 짧고 행동 중심으로 작성한다.

예시:

> 보행 주의 구간에 진입했습니다. 급경사와 전일 강수로 미끄럼 위험이 높습니다. 

## 5. AI/데이터 개선 방향

### 판단 원칙: 안전기준과 추천기준 분리

ForestRx의 판단 구조는 "먼저 안전 여부를 판단하고, 그다음 추천 순위를 정한다"로 고정한다.

1. 안전 등급 판단: 추천(그린), 주의(노랑), 비추천(빨강)
2. 등급별 정렬: 그린 코스를 우선 노출하고, 노랑은 주의 안내와 함께 보조 추천한다. 빨강은 원칙적으로 추천 목록에서 제외한다.
3. 추천 점수 정렬: 같은 안전 등급 안에서 시간, 접근성, 체력, 목적 적합도 기준으로 순위를 매긴다.

즉, 안전 등급이 추천 점수보다 항상 우선한다.

### 1단계: 규칙 기반 안전 등급 모델

현재 구현된 날씨 안전도, 일몰 여유, 난이도, 고도 상승량 점수를 확장한다.

- 강수량: 비가 오거나 전일 강수량이 있으면 암반/급경사 구간 위험 증가
- 풍속: 능선/정상부 위험 증가
- 일몰: 예상 하산 시간이 일몰에 가까우면 위험 증가
- 동반인: 어린이/노년층이 있으면 난이도, 거리, 고도 페널티 강화
- 지형: 급경사, 암반, 계단, 계곡 인접 구간에 위험 가중치 부여

### 안전 등급 산정 기준

안전 등급은 사용자가 실제로 산행해도 되는지를 판단하는 기준이다. 화면에는 점수나 퍼센트를 노출하지 않고 추천(그린), 주의(노랑), 비추천(빨강)만 보여준다.

#### 추천(그린)

- 강수 없음 또는 미미함
- 풍속 낮음
- 일몰 전 하산 여유 충분
- 사용자의 선택 시간 안에 무리 없이 완료 가능
- 난이도와 고도 상승량이 사용자 상태에 적합
- 보행 취약자 동반 시에도 급경사, 암반, 긴 코스 위험이 낮음

#### 주의(노랑)

- 약한 비 또는 전일 강수로 미끄럼 가능성 있음
- 풍속이 다소 높음
- 일몰 전 하산 여유가 30~60분 정도로 짧음
- 선택 시간보다 약간 긴 코스
- 고도 상승량이 크거나 일부 급경사 구간 있음
- 어린이/노년층 동반 시 보호자 주의 필요

#### 비추천(빨강)

- 강수량 높음
- 강풍
- 일몰 전 하산이 불가능하거나 여유가 30분 미만
- 선택 시간보다 지나치게 긴 코스
- 보행 취약자 동반인데 난이도 높음
- 급경사, 암반, 계곡 인접 위험 구간이 많음
- 산불 위험 또는 기상 특보 수준의 위험 존재

### 내부 안전 점수 가중치

내부 모델은 점수를 계산할 수 있지만 화면에는 노출하지 않는다. 점수는 등급 산출과 근거 생성에만 사용한다.

- 기상 안전성: 30
- 지형/코스 난이도: 25
- 일몰/시간 여유: 20
- 사용자 상태/보행 취약자 여부: 15
- 사고 이력/위험 구간: 10

룰 기반 게이트도 함께 적용한다.

- 빨강 조건이 하나라도 강하면 비추천
- 노랑 조건이 2개 이상이면 주의
- 위험 조건이 없고 시간/체력/기상 조건이 안정적이면 추천

### 추천 정렬 기준

추천기준은 "안전한 후보 중에서 어떤 코스를 더 위에 보여줄지"를 정하는 기준이다.

정렬 우선순위:

1. 안전 등급
2. 시간 적합도
3. 접근성
4. 체력/난이도 적합도
5. 사용자 목적
6. 데이터 신뢰도
7. 혼잡도

추천 점수 가중치:

- 시간 적합도: 25
- 접근성: 20
- 체력/난이도 적합도: 20
- 목적 적합도: 15
- 안전 여유도: 10
- 데이터 품질: 5
- 혼잡도: 5

안전은 이미 1차 필터로 작동하므로 추천 점수 안에서는 보조 지표로만 사용한다.

### 시간 기준

사용자가 입력한 시간은 정확한 절단값이 아니라 후보 탐색 기준이다.

- 선택 시간 1시간: 45~90분
- 선택 시간 2시간: 90~150분
- 선택 시간 3시간: 150~210분
- 선택 시간 4시간: 210~270분

보행 취약자 동반 시 상단 버퍼를 줄인다.

- 일반 사용자 2시간: 90~150분
- 보행 취약자 동반 2시간: 90~135분

### 등급별 설명 문장

추천(그린):

> 현재 기상과 하산 시간을 고려했을 때 안정적으로 이용 가능한 코스입니다.

주의(노랑):

> 일부 급경사 구간과 일몰 여유 부족이 있어 보호자 동반 확인이 필요합니다.

비추천(빨강):

> 현재 강수와 코스 난이도를 고려하면 보행 취약자 동반 산행에는 적합하지 않습니다.

### 2단계: 데이터 융합형 안전 등급 예측

대회 출품 설명에서는 아래 데이터를 융합하는 구조로 제시한다.

- 국립공원공단/산림청 탐방로 데이터: 코스, 거리, 난이도, 위치
- 기상청 단기예보: 강수, 풍속, 기온
- 한국천문연구원 일출/일몰: 시간 안전성
- 소방청 산악 구조 출동 이력: 사고 다발 구간
- 국토부 DEM/정밀 지형 공간정보: 경사도, 고도 변화
- 산림청 안전 매뉴얼: RAG 기반 안전 브리핑 지식

### 3단계: RAG/LLM 안전 에이전트

LLM은 추천 자체를 결정하지 않고, 산출된 안전 등급과 근거 데이터를 사용자 언어로 설명한다.

예시:

> 어제 강수와 현재 풍속을 고려하면 암반 구간 미끄럼 위험이 높습니다. 어린이 동반 산행에는 급경사 구간을 피하고 완만한 우회 코스를 추천합니다.

## 6. 필요 데이터 정의

추천기준과 안전기준을 계산하기 위해 필요한 데이터는 필수 데이터, 안전 고도화 데이터, Safe Link 데이터, B2G 확장 데이터로 나눈다.

### 필수 데이터

MVP에서 반드시 필요한 데이터다. 현재 구현된 추천 API와 가장 직접적으로 연결된다.

| 데이터 | 주요 필드 | 활용 목적 | 우선순위 | MVP 대체 방식 |
| --- | --- | --- | --- | --- |
| 탐방로/코스 데이터 | 산 이름, 코스명, 거리, 예상 소요 시간, 난이도, 위치 좌표, 고도 상승량 | 코스 후보 생성, 시간 적합도, 체력/난이도 적합도 계산 | P0 | 국립공원공단 탐방로 CSV/API + 부족 필드는 Mock 보강 |
| 현재 위치 데이터 | 사용자 위도, 경도, 위치 수집 시각 | 주변 코스 탐색, 접근성 계산, Safe Link 위치 기준 | P0 | 브라우저 Geolocation API |
| 기상 데이터 | 기온, 강수량, 풍속, 습도, 예보 기준 시각 | 추천/주의/비추천 안전 등급 산정 | P0 | 기상청 단기예보 API, 실패 시 Mock 날씨 |
| 일출/일몰 데이터 | 일몰 시각, 날짜, 위치 | 하산 가능 시간, 일몰 전 여유 시간 계산 | P0 | 한국천문연구원 API 또는 고정 Mock |
| 사용자 입력 데이터 | 이동 가능 시간, 등산 소요 희망 시간, 동반인, 체력/경험, 목적 | 안전 등급 가중치, 추천 정렬, 시간 버퍼 적용 | P0 | 앱 입력 폼 |

### 안전 고도화 데이터

대회 차별성을 강화하는 데이터다. 실제 연동이 어렵다면 MVP에서는 위험 구간 Mock 데이터로 시연한다.

| 데이터 | 주요 필드 | 활용 목적 | 우선순위 | MVP 대체 방식 |

### 출처별 개발 메모

- 공공데이터포털 API는 대부분 서비스키가 필요하다. 개발계정은 자동 승인인 경우가 많지만, 운영계정 트래픽은 활용사례 등록이 필요할 수 있다.
- `국립공원공단_탐방로_20240911`은 현재 프로젝트 루트의 CSV와 직접 연결되는 1순위 데이터다.
- 기상청 단기예보는 위도/경도 그대로 조회하는 방식이 아니라 격자 좌표 변환이 필요하다. 현재 백엔드의 날씨 API 래퍼에서 이 변환 책임을 가진다.
- 한국천문연구원 출몰시각 정보는 XML 기반이므로 백엔드에서 XML 파싱 후 `sunset`만 표준 필드로 변환한다.
- DEM은 대용량 GIS 파일이므로 MVP에서는 바로 연동하지 않고, 코스별 `slope_level`, `elevation_gain_m`, `risk_points` Mock 필드로 대체한다.
- 산악 사고 이력은 공개 API로 바로 쓰기 어려울 수 있다. 먼저 소방청 통계/보도자료로 근거를 잡고, 발표에서는 사고 이력 데이터 융합 구조를 제시한다.
- Safe Link 위치 데이터는 외부에서 받는 공공데이터가 아니라 ForestRx 앱에서 사용자 동의 후 직접 생성하는 1차 데이터다.

## 7. UI/UX 개선 방향

### 홈/입력 화면

- "오늘 코스 추천"보다 "오늘의 안전 산행 진단"을 전면에 둔다.
- 이동 가능 시간과 등산 소요 시간은 1시간 단위 선택지를 기본으로 제공한다.
- 사용자가 2시간을 선택해도 정확히 120분 코스만 찾지 않고, 백엔드에서 전후 30분 버퍼를 적용한다.

### 추천 결과 화면

- 코스 카드의 1순위 지표를 추천 점수보다 안전 등급이 높은 것을 기준으로 한다. 등급이 같으면 추천 점수로 순위를 매긴다. 
- 카드에 반드시 노출할 정보:
  - 안전 등급: 추천(그린), 주의(노랑), 비추천(빨강)
  - 주요 위험 요인
  - 우회 필요 여부
  - 취약자 적합 여부
- 리스트 상단에 AI 안전 브리핑 카드를 배치한다.

### 안전 등급 표현 원칙

- 퍼센트 표기는 사용하지 않는다. 73%처럼 보이는 수치는 근거보다 정밀해 보일 수 있어 사용자 판단을 흐린다.
- 사용자는 행동을 결정해야 하므로 등급은 3단계로 단순화한다.
- 추천(그린): 원활한 산행이 가능하다.
- 주의(노랑): 특정 구간이나 조건에 주의가 필요하며 우회/시간 조정이 권장된다.
- 비추천(빨강): 산행에는 부적합하며 대체 코스가 필요하다.
- 내부 모델은 점수를 계산할 수 있지만, 사용자 화면과 API의 핵심 표현은 등급과 근거 문장으로 제공한다.

### 모두 비추천일 때의 예외 화면

기상 악화, 일몰 임박, 강풍, 사용자 조건 부적합으로 주변 코스가 모두 비추천(빨강)일 수 있다. 이 경우 빈 화면이나 무리한 추천을 보여주지 않고 AI 안전 에이전트가 대체 행동을 제안한다.

표시 원칙:

- 추천 가능한 안전 코스가 없음을 명확히 알린다.
- 현재 반경과 판단 근거를 함께 보여준다.
- 등산 강행을 유도하지 않는다.
- 일정 연기, 검색 반경 확대, 쉬운 산책 코스, 실내 활동 대체를 제안한다.

예시 문구:

> 현재 반경 10km 내 안전한 코스가 없습니다. 강수와 일몰 여유 부족으로 대부분의 코스가 비추천 상태입니다. 오늘은 산행을 연기하거나 실내 활동을 추천합니다.

대체 CTA:

- 다른 날짜로 다시 보기
- 검색 반경 넓히기
- 쉬운 산책 코스만 보기
- 실내 대체 활동 보기

### 지도 상세 화면

- 지도는 단순 위치 표시가 아니라 위험 구간 시각화가 핵심이다.
- 안전 구간: 녹색 실선
- 주의 구간: 노란색 점선
- 위험 구간: 빨간색 점선
- 위험 마커: 급경사, 암반, 계단, 계곡, 일몰 위험, 강풍 위험
- 상세 하단에는 "왜 위험한가"와 "어떻게 우회하는가"를 짧게 보여준다.
- 산행 시작 후에는 세이프 링크 공유 상태, 보호자 연결 여부, 마지막 위치 동기화 시간을 표시한다.

### 세이프 링크 보호자 화면

- 보호자 화면은 복잡한 기능보다 모니터링에 집중한다.
- 상단에는 현재 상태를 "정상 이동", "경로 이탈 주의", "위험 구간 진입"으로 표시한다.
- 지도에는 대상자의 현재 위치와 이동 궤적을 크게 보여준다.
- 위험 알림 이력과 마지막 GPS 갱신 시간을 함께 제공한다.
- 긴급 상황에서는 전화 연결 또는 119 신고 안내 CTA를 제공한다.

## 8. 백엔드 로직 보완

### 시간 입력 버퍼

사용자는 이동 가능 시간과 등산 소요 시간을 1시간 단위로 입력하지만, 실제 코스 데이터는 1시간 45분, 2시간 30분처럼 세분화되어 있다. 따라서 백엔드는 선택 시간을 고정값으로 필터링하지 않고 버퍼 범위를 적용한다.

- 기본 버퍼: 선택 시간 전후 30분
- 예시: 사용자가 2시간을 선택하면 90~150분 코스를 후보로 포함한다.
- 보행 취약자 동반 시 상단 버퍼를 줄여 과도하게 긴 코스가 추천되지 않도록 한다.
- 버퍼 범위 안에서는 안전 등급을 먼저 보고, 등급이 같을 때 추천 점수로 정렬한다.

### 모두 비추천 응답

추천 후보가 모두 비추천이면 백엔드는 빈 리스트 대신 별도 상태를 반환한다.

권장 응답 필드:

- `result_state`: `has_recommendations` 또는 `no_safe_course`
- `safety_label`: `비추천`
- `agent_briefing`: 대체 행동 안내 문장
- `alternative_actions`: 다른 날짜, 반경 확대, 쉬운 코스, 실내 활동

### Safe Link API 범위

Kakao Maps SDK 내장 방식(A안)을 선택할 경우 아래 API를 MVP 후보로 둔다. 외부 링크 방식(B안)을 선택하면 세션 생성과 공유 링크만 남기고 위치 업로드/알림 API는 제외한다.

- `POST /api/safe-links/`: 위치 공유 세션 생성
- `POST /api/safe-links/{id}/locations/`: 위치 좌표 업로드
- `GET /api/safe-links/{id}/status/`: 보호자용 현재 상태 조회
- `POST /api/safe-links/{id}/alerts/`: 위험 구간 진입/경로 이탈 알림 기록
